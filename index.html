01 — The Problem

Documentation is killing site engineers

Every day on an active construction site, a QS or site engineer is managing one of the most information-dense jobs in the world — material quantities, structural drawings, casting records, labour logs — often across multiple supervisors and with zero standardisation.

```
Real workflow — Ministry of Works, Nigeria

"I'd capture everything in phone notes on-site — quantities of columns cast, beams poured, stairs completed. Then I'd go back to the office and rewrite it all from scratch into reports. Not once. Multiple times. Each supervisor wanted their own version with no shared template. Same data, different documents, every single day."

This isn't a Nigerian problem. It's a global construction industry problem. And it is completely unsolved.

```

03 — The Use Case

Clicky as your site documentation co-pilot

```

Here is the exact workflow HeyClicky would transform:

1
End of casting — data is scattered across phone notes

Engineer has notes like: "Col C3 × 4, 300×300, floor 2. Beam B7 × 2, 230×450." Messy. Informal. Accurate.

2
Engineer opens laptop — Clicky sees the screen

Clicky sees the phone notes or typed raw data on screen. Engineer says: "Clicky, turn this into a daily casting report."

3
Clicky generates the structured report instantly

Formatted table: element type, grid reference, dimensions, quantity, floor level, date. Professional. Ready to submit.

4
Multiple supervisors? One command.

"Clicky, now reformat this for the Project Manager's weekly summary." Same data. Different structure. Done in seconds.

5
Cumulative tracking over time

Clicky agent tracks totals across days — total columns cast this week, variance against programme, running material tally.

```

05 — Feature Breakdown

What needs to be built

```
🗣️
Voice-to-report

Dictate raw site notes out loud. Clicky structures them into a formatted casting or material record automatically.

📋
Multi-format output

Same underlying data → reformatted per supervisor preference. Engineer defines each supervisor's format once.

📊
Cumulative tracking agent

Background agent accumulates daily records into weekly and monthly summaries. Flags variance against programme.

📐
Drawing-aware Q&A

Clicky sees a structural PDF open on screen. Engineer asks "how many columns on this floor?" — Clicky reads and answers.
```
